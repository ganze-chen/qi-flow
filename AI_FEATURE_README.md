# AI 共创功能使用说明

## 功能概述

AI 共创功能使用 Google Gemini 3 Pro Image Preview 模型对导出的墨迹图像进行艺术化润色，将原始墨点转化为吴冠中风格的山水意境画。

## 启动步骤

### 1. 启动后端服务

在项目根目录打开终端，运行：

```bash
npm run server
```

服务将在 `http://localhost:3001` 启动。

### 2. 启动前端开发服务器

在另一个终端窗口运行：

```bash
npm run dev
```

前端将在 `http://localhost:3000` 启动。

### 或者一键启动（Windows）

```bash
npm run dev:full
```

这会同时启动后端和前端服务。

## 使用流程

1. **完成习练** - 跟随引导完成一次墨迹创作
2. **点击导出** - 在完成界面点击"导出"按钮
3. **查看预览** - 系统会自动显示原始图像
4. **AI 润色** - 勾选"AI 艺术润色"选项（默认开启）
5. **等待处理** - 系统会显示"AI 润色中..."加载动画（约 10-30 秒）
6. **查看结果** - 润色完成后自动显示增强后的图像
7. **保存作品** - 点击"保存"按钮下载图像

## 功能特性

- **自动增强** - 导出时默认启用 AI 润色
- **可选关闭** - 取消勾选"AI 艺术润色"可使用原始图像
- **实时预览** - 润色完成后立即在预览界面显示
- **错误处理** - 如果 AI 服务不可用，自动回退到原始图像
- **吴冠中风格** - 使用特定提示词生成极简抽象的山水意境

## 技术架构

```
前端 (Vite + Vanilla JS)
    ↓ HTTP POST
后端服务 (Express.js)
    ↓ API Call
Google Gemini 3 Pro Image Preview
    ↓ 返回增强图像
前端显示
```

## API 配置

API 密钥已在 `server.js` 中配置。如需更换：

1. 打开 `server.js`
2. 修改第 13 行的 `apiKey` 值
3. 重启后端服务

## 故障排查

### 问题：AI 润色失败

**解决方案：**
1. 确认后端服务已启动（检查 `http://localhost:3001/api/health`）
2. 检查控制台错误信息
3. 确认 API 密钥有效且有足够配额
4. 检查网络连接

### 问题：加载时间过长

**原因：** Gemini API 处理图像需要时间（通常 10-30 秒）

**建议：** 耐心等待，或取消勾选 AI 润色选项使用原始图像

### 问题：图像质量不理想

**调整方案：**
1. 打开 `src/core/AIEnhancer.js`
2. 修改第 5 行的 `DEFAULT_PROMPT` 提示词
3. 重新加载页面测试

## 文件结构

```
qi-flow-week5/

├── src/
│   └── core/
│       └── AIEnhancer.js        # AI 增强模块
└── src/main.js                  # 集成 AI 功能的主应用
```

## 注意事项

- 后端服务必须在前端之前启动
- API 调用会消耗 Gemini API 配额
- 图像处理时间取决于网络速度和 API 响应时间
- 建议在稳定网络环境下使用
